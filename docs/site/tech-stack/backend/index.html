    Backend¶

    Our server-side technology stack.

    FastAPI¶

    FastAPI is our web framework of choice.

    Why FastAPI?¶

    Feature Benefit
    Type hints Automatic validation, documentation, and IDE support
    Async support Native async/await for I/O-bound operations
    Auto documentation OpenAPI/Swagger generated from code
    Performance One of the fastest Python frameworks
    Pydantic integration Robust data validation

    Project Structure¶

    app/
├── apis/
│   └── v1.py              # API router with versioning
├── core/
│   ├── config.py          # Settings management
│   ├── database.py        # Database connection
│   ├── dependencies.py    # DI containers
│   └── security.py        # Auth utilities
├── models/                # SQLAlchemy models
├── schemas/               # Pydantic schemas
├── services/              # Business logic
├── crud/                  # Database operations
└── main.py               # Application entry

    Example Endpoint¶

    from fastapi import APIRouter, Depends, HTTPException, status
from app.schemas.user import UserCreate, UserResponse
from app.services.user import UserService
from app.core.dependencies import get_user_service

router = APIRouter(prefix="/users", tags=["users"])


@router.post("/", response_model=UserResponse, status_code=status.HTTP_201_CREATED)
async def create_user(
    data: UserCreate,
    service: UserService = Depends(get_user_service),
) -> UserResponse:
    """Create a new user."""
    user = await service.create(data)
    return user

    Python¶

    We use Python 3.11+ for all backend services.

    Version Policy¶

    • Minimum: Python 3.11
    • Target: Latest stable (currently 3.12)
    • Upgrade cycle: Within 6 months of new stable release

    SQLAlchemy 2.0¶

    We use SQLAlchemy's 2.0 style for ORM operations.

    Model Example¶

    from datetime import datetime
from uuid import UUID, uuid4
from sqlalchemy import String, DateTime, func
from sqlalchemy.orm import Mapped, mapped_column, relationship
from app.core.database import Base


class User(Base):
    __tablename__ = "users"

    id: Mapped[UUID] = mapped_column(primary_key=True, default=uuid4)
    email: Mapped[str] = mapped_column(String(255), unique=True, index=True)
    hashed_password: Mapped[str] = mapped_column(String(255))
    is_active: Mapped[bool] = mapped_column(default=True)
    created_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True), 
        server_default=func.now()
    )

    orders: Mapped[list["Order"]] = relationship(back_populates="user")

    Query Patterns¶

    from sqlalchemy import select
from sqlalchemy.ext.asyncio import AsyncSession


async def get_active_users(db: AsyncSession) -> list[User]:
    stmt = select(User).where(User.is_active == True)
    result = await db.execute(stmt)
    return result.scalars().all()

    Celery¶

    Background task processing with Celery and Redis.

    Use Cases¶

    • Sending emails
    • Processing uploads
    • Scheduled jobs (with Celery Beat)
    • Long-running computations

    Task Example¶

    from celery import shared_task
from app.services.email import send_email


@shared_task(
    bind=True,
    max_retries=3,
    default_retry_delay=60,
)
def send_welcome_email(self, user_id: str) -> None:
    """Send welcome email to new user."""
    try:
        user = get_user(user_id)
        send_email(
            to=user.email,
            subject="Welcome!",
            template="welcome",
        )
    except TransientError as e:
        raise self.retry(exc=e)

    Configuration¶

    # celery.py
from celery import Celery

app = Celery("tasks")
app.config_from_object({
    "broker_url": "redis://localhost:6379/0",
    "result_backend": "redis://localhost:6379/1",
    "task_serializer": "json",
    "result_serializer": "json",
    "accept_content": ["json"],
    "timezone": "UTC",
    "task_track_started": True,
})

    Pydantic¶

    Data validation and serialization.

    Schema Example¶

    from datetime import datetime
from uuid import UUID
from pydantic import BaseModel, EmailStr, Field


class UserBase(BaseModel):
    email: EmailStr
    full_name: str = Field(..., min_length=1, max_length=100)


class UserCreate(UserBase):
    password: str = Field(..., min_length=8)


class UserResponse(UserBase):
    id: UUID
    is_active: bool
    created_at: datetime

    model_config = {"from_attributes": True}

    Validation Features¶

    • Email validation with EmailStr
    • Length constraints with Field
    • Custom validators with @field_validator
    • Computed fields with @computed_field
